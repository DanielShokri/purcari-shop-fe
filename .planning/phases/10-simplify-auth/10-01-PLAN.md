## Current Problems Identified

1. **Double user creation**: 
   - `auth.ts` `createOrUpdateUser` callback creates user in `users` table
   - `useAuth.ts` `signUp`/`signIn` then calls `createOrUpdateUserProfile` mutation - REDUNDANT
   - This can cause race conditions and duplicate entries

2. **InvalidSecret error on password sign-in**:
   - Possibly from corrupted password hashes on existing accounts
   - May need to investigate if `AUTH_SECRET` env var is properly set
   - The validation in profile function might be throwing incorrectly

3. **Complex name validation in auth.ts profile**:
   - `fullNameRegex = /^\S{3,}\s+\S{3,}$/` requires "firstname lastname" format
   - This breaks for Google OAuth users with single names or different formats
   - Hebrew names may not match this pattern

4. **auth.config.ts may be unnecessary**:
   - Contains only `{ providers: [{ domain: CONVEX_SITE_URL, applicationID: "convex" }] }`
   - This appears to be legacy config from older @convex-dev/auth versions
   - Modern versions auto-detect these settings
</context>

       ```typescript
         };
       },
       ```

<task type="auto">
  <name>Task 2: Remove redundant createUserProfile call from useAuth.ts</name>
  <files>apps/storefront/hooks/useAuth.ts</files>
  <action>
    Remove the redundant `createUserProfile` mutation call from both `signUp` and `signIn` functions.

    The user is ALREADY created by `createOrUpdateUser` callback in auth.ts during the sign-in/sign-up flow. Calling `createOrUpdateUserProfile` afterward is:
    - Redundant (user already exists)
    - Can cause race conditions
    - Creates duplicate update queries
    - May be causing issues with existing accounts

    Changes to make:

    1. **Remove** the `createUserProfile` mutation import and usage:
       ```diff
       - const createUserProfile = useMutation(api.users.createOrUpdateUserProfile);
       ```

    2. **Remove** the `createUserProfile` call from `signUp`:
       ```diff
         await convexSignIn("password", {
           email: data.email,
           password: data.password,
           name: data.name,
           flow: "signUp",
         });
       -
       - await createUserProfile({
       -   phone: data.phone,
       -   name: data.name,
       -   email: data.email,
       - });
       ```

    3. **Remove** the `createUserProfile` call from `signIn`:
       ```diff
         await convexSignIn("password", {
           email: data.email,
           password: data.password,
           flow: "signIn",
         });
       -
       - await createUserProfile({
       -   phone: "", 
       -   email: data.email,
       - });
       ```

    4. **Keep** the `signInWithGoogle` function unchanged (it doesn't call createUserProfile)

    Note: Phone number update should be handled by a separate `updateProfile` mutation if needed, not during auth flow.
  </action>
  <verify>
